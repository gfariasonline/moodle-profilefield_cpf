```
тФВ   тФФтФАтФА build/                   # Generated by grunt тАФ ALL files MUST be committed

```

**Why `local/` subdirectory for cleave.js:**  
Moodle convention (`media/player/videojs/amd/src/local/`) for third-party libraries that are not Moodle-authored modules. It visually separates vendor files from plugin code. The grunt build picks it up via `amd/src/**/*.js` and builds it to `amd/build/local/cleave.min.js`.

## Module Pattern

### cpf_mask.js тАФ Use AMD `define()`, not ESM `import`

```js
// This file is part of Moodle - http://moodle.org/
// ... GPL header ...

/**
 * CPF input masking using cleave.js.
 *
 * @module     profilefield_cpf/cpf_mask
 * @copyright  2026 Author
 * @license    http://www.gnu.org/copyleft/gpl.html GNU GPL v3 or later
 */
define(['./local/cleave'], function(Cleave) {
    return {
        init: function(fieldName) {
            new Cleave('#id_' + fieldName, {
                blocks: [3, 3, 3, 2],
                delimiters: ['.', '.', '-'],
                numericOnly: true
            });
        }
    };
});
```

**Why AMD `define()` and not ESM `import Cleave from './local/cleave'`:**  
Rollup processes each `amd/src/*.js` as an independent module. A static ESM import (`import X from './local/cleave'`) causes rollup to inline `cleave.js` directly into `cpf_mask.min.js` тАФ but rollup has no `@rollup/plugin-commonjs`, so it cannot correctly process the UMD wrapper. AMD `define(['./local/cleave'], ...)` is treated as a runtime dependency string, not a static bundling instruction. Rollup leaves it as-is. RequireJS resolves it at runtime.

This is exactly the pattern used by `filter/codehighlighter/amd/src/prism-init.js` (Moodle core, referencing a third-party `prism.js`).

### amd/src/local/cleave.js тАФ Copy UMD dist verbatim

Add license comment header at the very top (required by Moodle third-party lib conventions):

```js
/*!
 * cleave.js 1.6.0
 * https://github.com/nosir/cleave.js
 * License: MIT
 * Copyright (c) nosir
 */
(function webpackUniversalModuleDefinition(root, factory) {
    // ... rest of cleave.js UMD dist unchanged ...
```

The Moodle grunt build (`babel-plugin-add-module-to-define.js`) finds the `define([], factory)` call inside the UMD wrapper and injects the module name: `define ('profilefield_cpf/local/cleave', [], factory)`. This makes RequireJS register it correctly at runtime.

### thirdpartylibs.xml тАФ Required at plugin root

```xml
<?xml version="1.0"?>
<libraries>
    <library>
        <location>amd/src/local/cleave.js</location>
        <name>cleave.js</name>
        <description>Format input text content when typing. Used for CPF masking.</description>
        <version>1.6.0</version>
        <license>MIT</license>
        <repository>https://github.com/nosir/cleave.js</repository>
        <copyrights>
            <copyright>nosir</copyright>
        </copyrights>
    </library>
</libraries>
```

`grunt ignorefiles` (which runs first as part of `grunt amd`) reads this file and adds `amd/src/local/cleave.js` to `.eslintignore`. Without this, ESLint will fail on third-party code that doesn't follow Moodle style.

## Build Commands

### Prerequisites

Node.js 22.x must be active. Using the local Moodle installation's `.nvmrc`:

```bash
cd /path/to/moodle45
nvm use  # reads .nvmrc = lts/jod (Node 22.x)
```

Moodle root `node_modules` must be installed (only needed once):

```bash
cd /path/to/moodle45
npm install
```

### Running the AMD Build

**Option A тАФ from the plugin's `amd/` directory (scoped, recommended for development):**

```bash
cd /path/to/moodle45/user/profile/field/cpf/amd
/path/to/moodle45/node_modules/.bin/grunt amd
```

Grunt finds `Gruntfile.js` by traversing up the directory tree to Moodle root. The component is discovered via filesystem glob on `user/profile/field/*/version.php` тАФ no Moodle admin install needed.

**Option B тАФ from Moodle root (processes all plugins):**

```bash
cd /path/to/moodle45
npx grunt amd
```

This rebuilds AMD modules for ALL Moodle components. Slower but equivalent.

### What `grunt amd` Does

1. `ignorefiles` тАФ reads all `thirdpartylibs.xml` files and writes `.eslintignore`
2. `eslint:amd` тАФ lints all `amd/src/**/*.js` excluding ignored paths
3. `rollup` тАФ for each `amd/src/**/*.js`, runs:
   - Babel (`transform-es2015-modules-amd-lazy` + `babel-plugin-add-module-to-define.js`) to convert ES6/AMD to named AMD modules
   - Terser (`mangle: false`) to minify
   - Outputs `amd/build/X.min.js` and `amd/build/X.min.js.map`

### Commit the Build Output

After running grunt:

```bash
git add amd/build/cpf_mask.min.js amd/build/cpf_mask.min.js.map
git add amd/build/local/cleave.min.js amd/build/local/cleave.min.js.map
```

**This is mandatory.** moodle-plugin-ci grunt:
1. Backs up the plugin
2. Deletes `amd/build/`
3. Runs `grunt amd`
4. Compares every `.js` and `.js.map` file in backup vs. rebuilt output
5. **FAILS** if any file is stale (`sha1` differs), missing, or newly generated

## Alternatives Considered

| Recommended | Alternative | Why Not |
|-------------|-------------|---------|
| AMD `define()` in cpf_mask.js | ESM `import Cleave from './local/cleave'` | Rollup would inline cleave.js into cpf_mask.min.js тАФ works only with `@rollup/plugin-commonjs` which Moodle's build does NOT include |
| UMD dist (`dist/cleave.js`) | ESM dist (`dist/cleave-esm.js`) | ESM dist via static import causes rollup to inline it AND still produce a separate `cleave.min.js` (double-bundling). Unnecessary complexity |
| Local UMD copy in `amd/src/local/` | Load from CDN | PROJECT.md explicitly forbids CDN; Moodle installations are often offline |
| Moodle root's node_modules + Gruntfile | Plugin-level Gruntfile.js + package.json | Plugin-level Gruntfile bypasses Moodle AMD conventions. moodle-plugin-ci uses plugin-dir Gruntfile.js only when it exists тАФ creates non-standard CI path |
| cleave.js 1.6.0 UMD | Custom input event handler | Custom handlers are fragile (cursor position, paste, mobile) тАФ the problem cleave.js already solves |

## What NOT to Use

| Avoid | Why | Use Instead |
|-------|-----|-------------|
| ESM static `import` for cleave.js | Rollup tries to bundle it inline; fails without `@rollup/plugin-commonjs`. See prism-init.js in Moodle core for the correct pattern | AMD `define(['./local/cleave'], ...)` |
| CDN load for cleave.js | Fails on offline Moodle installs; violates project requirement | Local copy in `amd/src/local/` |
| `amd/src/cleave.js` (top-level) vs `amd/src/local/cleave.js` | Both work; top-level is valid but Moodle convention uses `local/` for vendored code | `amd/src/local/cleave.js` |
| Dynamic `import('profilefield_cpf/local/cleave')` in cpf_mask.js | Unnecessarily async for synchronous masking init; fine but adds complexity | Synchronous AMD `define()` with static dependency array |
| Skipping `amd/build/` from git | moodle-plugin-ci grunt WILL FAIL тАФ it explicitly compares committed build files against freshly rebuilt output | Commit `amd/build/` after every `amd/src/` change |
| Running grunt with wrong Node.js version | Build output may differ from CI, causing "stale file" failures | `nvm use` from Moodle root before running grunt |

## moodle-plugin-ci Grunt Validation тАФ How It Fails

The `moodle-plugin-ci grunt` check (from `src/Command/GruntCommand.php`) does the following:

1. **Backs up** the entire plugin directory
2. **Deletes** `amd/build/` (the third argument to `GruntTaskModel`)
3. **Runs** `npx grunt amd` from `plugindir/amd/` working directory
4. **Compares** every `*.js` and `*.js.map` file:
   - File in backup but not rebuilt тЖТ "should be deleted" тЖТ EXIT 1
   - File in both but sha1 differs тЖТ "stale, needs rebuild" тЖТ EXIT 1
   - File rebuilt but not in backup тЖТ "newly generated, needs to be added" тЖТ EXIT 1
5. **Restores** plugin from backup

**Practical implication:** Any time you edit `amd/src/cpf_mask.js` or `amd/src/local/cleave.js`, you must run `grunt amd` and commit the updated `amd/build/` files in the same commit. Leaving build artifacts out of sync with source is the most common cause of this CI step failing.
